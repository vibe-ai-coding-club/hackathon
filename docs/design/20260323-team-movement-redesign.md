# 팀 이동 상호작용 리디자인

> 작성일: 2026-03-23
> 상태: 확정

## 요구 사항

### 변경 사항

1. **공개형 "팀원 구합니다" 기능 제거**
   - 참여자가 직접 팀에 들어가는 기능 차단
   - 현재의 recruiting sidebar에서 팀 클릭 → 이동 플로우 제거

2. **"팀 모집중" 섹션 → 디스코드 안내로 제한**
   - 모집 중인 팀 목록은 표시하되, 직접 이동 불가
   - "디스코드에서 팀원을 모집하세요" 안내 문구로 대체

3. **팀 → 팀원 초대 (단방향)**
   - 팀(리더)이 멤버를 초대하는 방식으로 변경
   - 참여자 본인이 팀에 합류하는 기능은 차단
   - 초대 시스템 (invite → accept/reject) 또는 즉시 이동 방식 선택 필요

4. **참여자 이동 시 팝업창**
   - 참여자가 수행하는 모든 이동은 팝업(모달)으로 구현 (기존 구현 유지)

5. **상태값 기반 이동 차단**
   - 팀/멤버 상태에 따라 이동 차단
   - 관리자만 임의 이동 가능 (기존 어드민 기능 유지)

6. **참여자 프로필 카드**
   - 대시보드 상단에 프로필 카드 배치
   - 표시 정보: 이름, 이메일, 전화번호, 경험 수준
   - 소속 팀 정보 + "팀 나가기" 버튼
   - 소속 팀 상태 표시/수정 (팀원 모집중 / 팀원 확정)
   - 내 상태 표시/수정 (팀 확정 / 팀 찾는중)
   - 정보 구성 자유롭게 가능
   - 팀에 소속된 참여자에게만 보임

## 확정된 상호작용 설계

### 1. 팀원 추가 (단방향 초대 — 즉시 이동)

- 팀 테이블의 **빈 팀원 슬롯 첫 번째 위치**에 "팀원 추가" 버튼 렌더링
- 버튼 클릭 → **모달/다이얼로그** 오픈
- 모달에 **구직중(seekingTeam=true) 참여자 목록** 표시
- 목록에서 클릭 → 즉시 초대 (기존 recruit API 재활용)
- 리더만 팀원 추가 버튼 사용 가능

### 2. 모집중 사이드바

- 모집중인 팀 목록은 **기존대로 노출**
- 클릭 시 팀 이동이 아닌 **디스코드로 팀장에게 문의** 안내
- 사이드바 안내 문구 변경

### 3. 프로필 카드

- 대시보드 상단에 **단순 노출** (이름, 이메일, 전화번호, 경험 수준, 소속 팀)
- **팀 모집 상태 토글**: 모집중 ↔ 모집 OFF (리더만)
- **개인 구직 상태 토글**: 팀 찾는중 ↔ OFF
- 팀 나가기 버튼 (기존 leave-modal 재사용)
- 개인 정보 수정은 미포함 (단순 표시)

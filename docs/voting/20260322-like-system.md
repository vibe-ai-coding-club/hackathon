# 좋아요(Like) & 투표 설정 리팩터링 설계

## 개요

발표 진행 중 참가자들이 실시간으로 프로젝트에 좋아요를 표현할 수 있는 기능을 추가한다.
동시에 기존 투표 시스템의 **세션 개념을 제거**하여, 참가자의 투표/좋아요 흐름을 방해하지 않도록 한다.

## 투표 vs 좋아요 비교

| 항목      | 투표(Vote)              | 좋아요(Like)                      |
| --------- | ----------------------- | --------------------------------- |
| 목적      | 우수상 점수 반영        | 발표 중 실시간 응원/반응          |
| 제한      | 1인당 N표 (관리자 설정) | 모든 프로젝트에 각 1회            |
| 자기 팀   | 투표 불가               | 제한 없음 (자기 팀도 좋아요 가능) |
| 차단 여부 | 차단 없음, 항상 가능    | 차단 없음, 항상 가능              |
| 실시간성  | 별도 실시간 표시 없음   | 실시간 카운트 표시 (폴링)         |

## 세션 → 설정 리팩터링

### 기존 문제

`VoteSession`이 `isActive`로 투표를 차단/허용하는 세션 라이프사이클을 가짐.
→ 참가자가 원할 때 투표할 수 없고, 관리자가 시작/종료를 제어해야 하는 불필요한 병목.

### 변경 방향

- **세션 라이프사이클 제거**: `isActive`, `startedAt`, `endedAt` 제거
- **VoteSession → EventSetting**: 이벤트 전체 설정 테이블로 변경
  - `maxVotes`: 관리자가 언제든 변경 가능한 1인당 최대 투표 수
  - `presentingProjectId`: 현재 발표 중인 프로젝트
- 투표 API에서 세션 활성 여부 체크 로직 제거 → `maxVotes`만 확인

## 핵심 규칙

### 좋아요

| 항목         | 내용                                |
| ------------ | ----------------------------------- |
| 좋아요 자격  | 로그인한 참가자 (Member)            |
| 좋아요 수    | 프로젝트당 1회 (토글: 추가/취소)    |
| 자기 팀 제한 | 없음                                |
| 실시간 표시  | 폴링 기반으로 좋아요 수 실시간 갱신 |

### 발표 중 프로젝트

- 관리자가 현재 발표 중인 프로젝트를 선택
- 참가자 화면에서 해당 프로젝트가 **상단에 강조 표시**
- 나머지 프로젝트는 기존대로 그리드에 표시
- 관리자가 다른 프로젝트로 전환하거나 해제 가능

## 데이터 모델

### Like (신규)

```
Like
├── id
├── memberId     ← FK → Member
├── projectId    ← FK → Project
├── createdAt
└── UNIQUE(memberId, projectId)
```

### EventSetting (VoteSession 대체)

```
EventSetting
├── id
├── maxVotes              ← 1인당 최대 투표 수 (기본 5)
├── presentingProjectId   ← 현재 발표 중인 프로젝트 (FK → Project, nullable)
├── createdAt
├── updatedAt
```

- 싱글톤 행으로 운영 (1개만 존재)
- `isActive`, `startedAt`, `endedAt` 삭제

## API 설계

### 좋아요 API (신규)

| 메서드 | 경로        | 설명                |
| ------ | ----------- | ------------------- |
| GET    | `/api/like` | 내 좋아요 목록 조회 |
| POST   | `/api/like` | 좋아요 추가         |
| DELETE | `/api/like` | 좋아요 취소         |

### 프로젝트 목록 API (기존 확장)

| 메서드 | 경로                 | 변경사항                                       |
| ------ | -------------------- | ---------------------------------------------- |
| GET    | `/api/vote/projects` | 응답에 `likeCount`, `presentingProjectId` 추가 |

### 관리자 API (변경)

| 메서드 | 경로                       | 변경사항                                |
| ------ | -------------------------- | --------------------------------------- |
| GET    | `/api/admin/event-setting` | 현재 설정 + 투표/좋아요 결과 조회       |
| POST   | `/api/admin/event-setting` | maxVotes 변경, presentingProjectId 변경 |

### 투표 API (변경)

| 메서드 | 경로        | 변경사항                              |
| ------ | ----------- | ------------------------------------- |
| POST   | `/api/vote` | 세션 활성 체크 제거 → maxVotes만 확인 |
| DELETE | `/api/vote` | 세션 활성 체크 제거                   |

## UI 변경

### 참가자 UI (`/teams/vote`)

- 발표 중 프로젝트: 상단에 별도 섹션으로 강조 표시, 좋아요 버튼 + 실시간 카운트
- 나머지 프로젝트: 기존 그리드, 각 카드에 좋아요 버튼 추가
- 투표 상태바: 세션 상태 표시 제거, 남은 투표 수만 표시
- 폴링 주기: 3초 간격으로 좋아요 수 + 발표 중 프로젝트 갱신

### 관리자 UI (`/admin/votes`)

- 세션 시작/종료 버튼 → maxVotes 조절 + 발표 중 프로젝트 선택으로 대체
- 좋아요 결과도 함께 표시
